2017년부터 2026년까지 9년간 개발을 해왔다. 프리랜서로 시작해서 코인 선물 거래소, 호주 온라인 카지노 외주 개발을 하다가 인플루언서 플랫폼을 창업했다. 이후 공동주택 관리 서비스, 학원 ERP까지. 그런데 돌이켜보니 묘한 패턴이 있었다. 창업한 회사는 수익성이 낮아서 접었고, 취직한 스타트업들은 서비스 인프라가 안정될 때쯤 자금난으로 월급을 못 주겠다고 했다.

## 반복되는 패턴

### 창업: 인플루언서 플랫폼

직접 창업했다. 블로거 영향력 점수 알고리즘을 만들었다. 크롤링, 검색 순위 분석, Redis 캐싱까지 구축했다. 광고주들 반응도 좋았다.

**기술적으로는 안정화됐다:**
- 블로거 1,000명 분석 가능
- Redis 캐싱으로 비용 절감
- API 응답 속도 0.3초

**그런데:**
- 매출은 나왔지만 너무 적었다
- 일하는 양에 비해 수익이 적었다
- 이대로는 성장이 어렵다고 판단

더 큰 곳에서 경험을 쌓고 싶어서 회사를 정리하고 취직했다.

### 첫 번째 취직: 공동주택 관리 서비스

PHP 레거시를 FastAPI로 재개발했다. 3개월 만에 모든 기능 이전했다. 트래픽도 문제없이 처리했다.

**기술적으로는 성공했다:**
- 20개 단지, 수천 세대 서비스
- API 응답 시간 50% 개선
- 주니어 개발자 온보딩도 순조로움

**그런데:**
- 결제 처리 금액은 컸지만 수수료가 낮았다
- 회사가 실제로 가져가는 돈은 거의 없는 구조
- 6개월 후 자금난

또 나왔다.

### 두 번째 취직: 학원 ERP

크론 서버 모니터링 자동화, LLM 에러 자동 수정 파이프라인까지 구축했다. 장애 대응 시간 50% 단축했다.

**기술적으로는 잘 돌아갔다:**
- 5-60개 크론잡 안정 운영
- PHP Fatal Error 자동 수정
- 운영 공수 70% 절감
- 월 40억 원 결제 처리

**그런데:**
- 월 40억 처리했지만 회사가 가져가는 건 거의 없었다
- 결제 중개 수수료만으로는 회사 운영이 어려웠다
- 다시 자금난

패턴이 보였다.

## 깨달은 것

### 1. 기술은 필요조건이지 충분조건이 아니다

인프라가 안정되면 서비스가 잘 돌아간다. 그런데 그게 매출로 이어지지 않는다.

- API 응답 속도 0.3초? 사용자는 1초도 괜찮다고 생각한다
- 장애 대응 시간 50% 단축? 고객사는 장애가 없는지만 관심있다
- LLM 자동화? 영업팀은 신규 고객 확보에 더 신경쓴다

기술로 해결할 수 있는 건 많다. 하지만 비즈니스 문제는 기술만으로 안 풀린다.

### 2. 스타트업은 타이밍 싸움이다

내가 인프라를 안정화할 때쯤 회사 자금이 바닥났다. 이유는 간단하다.

**초기 6개월:**
- 기능 개발에 집중
- 버그 투성이지만 일단 작동
- 고객사 확보 집중

**다음 6개월:**
- 인프라 안정화 시작
- 버그 수정, 모니터링 구축
- 기술 부채 해결

**그 다음:**
- 인프라는 안정됨
- 그런데 매출은 그대로
- 자금 바닥

인프라를 안정화하는 동안 영업은 멈춰있었다. 개발팀이 안정화에 집중하는 사이, 회사는 돈이 떨어져갔다.

### 3. 월급 지연은 신호다

"이번 달만 참아달라", "투자 유치되면 바로 줄게"

듣기 좋은 말이다. 그런데 한 번 밀리면 계속 밀린다.

**내가 본 패턴:**
- 1개월 지연: "죄송합니다, 다음 달엔 드릴게요"
- 2개월 지연: "투자 유치 직전이에요, 조금만 기다려주세요"
- 3개월 지연: "회사가 어려워졌습니다"

월급이 한 번 밀리면 회사가 회복하는 경우를 못 봤다. 빨리 나오는 게 답이다.

### 4. 외국 경험도 비슷했다

호주 온라인 카지노 개발도 해봤다. 기술적으로는 재밌었다.

- 실시간 배팅 처리
- 고액 트래픽 대응
- 결제 시스템 연동

그런데 여기도 마찬가지였다. 규제 변경, 라이센스 문제로 서비스가 흔들렸다. 기술이 좋다고 비즈니스가 안정되는 건 아니었다.

## 배운 것

### 1. 회사를 볼 때 기술만 보지 마라

면접 갈 때 물어봐야 할 것:
- 현재 매출은? (결제 처리액이 아니라 실제 수익)
- 수익 구조는? (수수료율은 얼마나 되는지)
- 고객사 이탈률은?
- 신규 고객 확보 속도는?
- 런웨이(현금 소진 기간)는?

**중요한 깨달음:**
월 40억 결제 처리해도 회사가 망할 수 있다. 결제 처리액과 실제 수익은 다르다. 수수료가 1%면 40억 처리해도 회사가 가져가는 건 4,000만 원이다. 여기서 인건비, 서버비 빠지면 남는 게 없다.

기술 스택, 인프라, 개발 문화만 물어봤던 게 후회된다.

### 2. 기술 부채는 비즈니스 상황을 보고 해결해라

"기술 부채를 해결해야 한다"는 개발자의 당연한 생각이다. 그런데 회사 상황을 봐야 한다.

**회사에 돈이 있을 때:**
- 리팩토링 해라
- 인프라 개선해라
- 모니터링 구축해라

**회사에 돈이 없을 때:**
- 최소한만 고쳐라
- 매출에 직접 영향 주는 것만 해라
- 기술 부채는 나중에

나는 회사 상황을 안 보고 기술 부채 해결에 몰두했다. 회사는 매출을 원했는데, 나는 안정성을 원했다.

### 3. 9년 동안 기술은 늘었다

회사는 망했지만 기술은 늘었다.

**2017년:**
- PHP, Python 기본 문법
- MySQL 쿼리 작성
- 프리랜서로 외주 개발

**2026년:**
- FastAPI, Django, Spring Boot 경험
- Redis 캐싱, 쿼리 튜닝
- CI/CD, 모니터링 구축
- LLM 자동화 파이프라인

회사가 망해도 기술은 남는다. 다음 회사에서 쓸 수 있다.

### 4. 스타트업은 도박이다

성공할 수도 있고 망할 수도 있다. 그런데 확률은 망하는 쪽에 가깝다.

**받아들여야 할 것:**
- 월급 지연 가능성
- 회사 폐업 가능성
- 스톡옵션 휴지 조각 가능성

**대비해야 할 것:**
- 비상금 6개월치
- 다른 회사 면접 준비
- 기술 블로그, GitHub 관리

스타트업 다니면서 "이 회사는 성공할 것 같아"라고 생각하지 마라. 언제든 나갈 준비를 해둬라.

## 지금

9년 동안 여러 회사를 다녔다. 성공한 회사는 없었다. 그래도 후회는 없다.

**기술적 성장:**
- 다양한 도메인 경험 (카지노, 인플루언서, 공동주택, 학원)
- 레거시 재개발 경험
- 스타트업 0→1 경험

**비즈니스 이해:**
- 기술만으로는 회사가 안 된다
- 매출이 없으면 아무것도 소용없다
- 타이밍이 중요하다

**현실적 교훈:**
- 월급 지연은 신호다
- 스타트업은 도박이다
- 회사는 망해도 기술은 남는다

## 마무리

"인프라가 안정될 때쯤 문 닫는 회사들"

이상하게 들릴 수 있지만, 실제로 내가 겪은 패턴이다. 기술적으로 잘 만들었는데 회사는 망했다.

그래도 계속 개발한다. 이제는 조금 더 현실적으로 본다. 기술도 중요하지만, 비즈니스가 더 중요하다. 회사를 볼 때 기술 스택만 보지 않는다. 매출, 고객사, 런웨이를 먼저 본다.

9년 차 개발자가 된 지금, 기술은 늘었지만 회사를 보는 눈도 생겼다. 다음 회사는 좀 더 오래 다닐 수 있을 것 같다. 아니면 또 6개월 만에 나올 수도 있다. 그것도 경험이다.
